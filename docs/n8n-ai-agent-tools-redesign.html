三個 Agent 工具化方案

A · Agent 外掛工具

工具都在 Agent 外面跑

先用 Switch 判斷要不要查 RAG、搜尋、圖片，整理好 context 後再丟給 Agent。

  • 最容易 debug
  • 節點會比較多
  • Agent 自主性較低
B · 推薦

Hybrid Agent Tools

穩定工具放外面預處理，RAG 搜尋、Brave Search、圖片等做成 Agent tools。主 Agent 可以自主決定何時用工具。

  • 畫布乾淨，仍可控
  • 保留全部功能
  • 少量 Code 只做格式兼容
C · Full Agent

所有能力都變成工具

入口之後幾乎直接進 Agent，所有記憶、搜尋、圖片、profile 都由 Agent tool 調用。

  • 最漂亮、最少節點
  • 測試和排錯較難
  • 對 n8n AI 節點能力要求最高

建議採用 B

因為你要「全部保留」而且可以接受少量必要 Code，Hybrid Agent Tools 最適合。它看起來會像現代 AI Agent workflow，但不會犧牲穩定性。

新版 workflow 畫布設計

畫布分成 5 區：入口路由、Agent Core、Agent Tools、資料/記憶、輸出與錯誤。每個區塊用 Sticky Note 固定邊界，主線從左到右，工具從下方接入 Agent。

入口
WhatsApp Webhook 接收 bridge 傳來的群訊息、私訊、圖片命令。
Normalize Payload Set / Edit Fields。整理 sender、group、text、messageId、timestamp。
Intent Router Switch。chat、record、memory、image、admin、health check。
入口區只做格式標準化，不做 AI 判斷。
Agent Core
DeepSeek Chat Model OpenAI-compatible Chat Model 或 DeepSeek 社區節點。
WhatsApp AI Agent 主決策節點。接收系統人格、輸入訊息、可用工具清單。
Agent Memory Instructions Prompt Template / Set。規則明確：可查記憶，不洩露原始歷史。
Structured Output Parser 把 Agent 回覆整理為 text、action、imagePayload、memoryAction。
大型 `prepare memory` 的 prompt 組裝會拆到 Agent prompt、tools 和 Set 節點。
Agent Tools
Tool: Search Memory Qdrant Vector Store，用現有 collection/schema 查相關對話。
Tool: Write Memory 保存 record 指令和重要對話摘要，沿用現有 Qdrant 資料。
Tool: Brave Search HTTP Request + n8n credential，替代 Code fetch。
Tool: Image Generate/Edit 圖片支線工具，回傳圖片 URL / binary payload。
Tool: Memory Status 回答記憶狀態、collection 檢查、簡單 admin 查詢。
工具區是新版的核心視覺亮點。
資料/記憶
Existing Qdrant Collection 不重建，不刪資料，先兼容現有 collection。
Member Profile Retriever 查 sender profile、group profile、常見稱呼。
Recent Context Store 保留近幾輪 AI turn，可用 Data Store 或 Qdrant metadata。
Compatibility Formatter 少量 Code，只處理現有資料 schema 差異。
資料區明確標示「沿用現有 Qdrant」。
輸出
Format WhatsApp Reply Set / Edit Fields。文字、圖片、引用、錯誤訊息格式。
Send to Bridge HTTP Request 回打本地 WhatsApp bridge API。
Save AI Turn 保存 AI 回覆摘要，提供近期上下文。
Error Fallback 任一工具失敗時，回覆簡短可理解錯誤。
輸出區統一回覆，不讓工具節點直接散落回 WhatsApp。

實作順序草案

建立全新 workflow

新命名為 `whatsapp bot AI Agent tools`，舊版只保留作備份。

先接通 Agent 主線

Webhook → Normalize → DeepSeek Chat Model → WhatsApp AI Agent → Send Reply。

加入現有 Qdrant 查詢工具

先只讀不寫，確認現有 collection/schema 可以被新版正確理解。

加入寫記憶、profile、近期上下文

拆成三個工具，不再混在單一 prepare memory code 中。

加入 Brave Search 和圖片工具

工具獨立測試，再接入 Agent。失敗時走 fallback。

整理畫布與驗證

Sticky Notes 分區、節點命名統一、測試每條 command route。
