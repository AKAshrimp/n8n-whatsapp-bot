n8n Community Node Redesign · Draft 01

把 WhatsApp AI Bot 重設計成更清楚、更好看的社區節點 workflow

目標不是在舊 workflow 上修補，而是新建一套乾淨的 n8n project。大 Code node 拆成標準節點、AI Agent、Qdrant、HTTP Request、Switch、Set/Edit Fields 等可視化模組，讓流程可以被看懂、維護和展示。

設計方向 New Project
核心原則 少 Code
視覺風格 分區清楚

我建議的新 workflow 視覺結構

把畫布分成 5 條泳道，每條泳道只做一類事情。這樣打開 n8n 時不用追一堆斜線，也能很快知道「訊息去哪裡」、「記憶去哪裡」、「AI 怎麼回」。

入口
When chat message received Webhook / Chat Trigger，接收 WhatsApp bridge 傳來的訊息。
Normalize Message Set / Edit Fields，統一 message、sender、group、timestamp。
Command Router Switch，分流 chat、record、memory、image、admin。
入口區保持短，方便 debug webhook payload。
記憶
Embeddings OpenAI-compatible Embeddings 或社區 embedding node。
Qdrant Vector Store 替代多個 build qdrant request Code node。
Member Profile Lookup Qdrant / Data Store 查群成員 profile 和近期上下文。
Memory Policy If / Switch 控制是否寫入、是否查詢、是否回覆記憶狀態。
保留 RAG，但把 JSON request builder 移走。
工具
Brave Search HTTP Request + Credential，替代 Code 裡讀 key 和 fetch。
List Collections Qdrant 管理節點，用於健康檢查和 collection debug。
Image Tool HTTP Request 或社區 image generation node，獨立成支線。
工具節點集中擺放，連到 AI Agent 或分支流程。
AI
DeepSeek Chat Model 用 n8n AI model node 或 OpenAI-compatible Chat Model。
Basic LLM Chain / AI Agent 主回覆生成，接收記憶、profile、web search context。
Structured Output Parser 取代 parse web search decision 這類解析 Code。
Reply Guardrails 簡短安全規則、不要洩露原始歷史、控制語氣。
AI 區只看模型、prompt、工具和輸出。
輸出
Format Reply Set / Edit Fields，把 AI output 整理成 WhatsApp bridge 格式。
Send WhatsApp Reply HTTP Request 回打本地 bridge API。
Save AI Turn Data Store / Qdrant，保存近期 AI 回覆上下文。
Error Reply Error Trigger / IF fallback，失敗時回簡短提示。
輸出區專管格式、回覆和錯誤。

舊版問題

prepare memory 太大 Qdrant request 由 Code 手寫 Brave Search key 和 fetch 混在 Code 裡 AI prompt、記憶、profile、工具耦合 畫布線路斜穿，不容易看懂

新版目標

每區 4 至 6 個主要節點 大部分邏輯用 Switch、If、Set、AI 節點表達 Credential 交給 n8n 管理，不在 Code 中讀檔 RAG、Web Search、Image 各自成工具支線 Sticky Note 分區，像產品架構圖一樣清楚

三個可選方案

A · 最穩

保守重排版

保留現在的 HTTP Request 和少量 Code，只把大 Code node 拆小、加註解和分區。最快落地，但不像真正社區節點化。

  • 風險最低
  • 能最快跑通
  • 視覺改善有限
B · 推薦

社區節點優先

新建 workflow，優先使用 AI Agent、Chat Model、Qdrant、HTTP Request、Set、Switch、If。只保留少數無法避免的小 Code。

  • 好看且可維護
  • 保留現有功能
  • 需要確認 n8n 版本和可安裝節點
C · 最乾淨

Agent 工具化重做

把 RAG、Search、Image 都做成 AI Agent tools，主流程只剩入口、Agent、輸出。畫布最漂亮，但要接受更多 LangChain/n8n AI 節點約束。

  • 最像現代 n8n AI workflow
  • 節點最少
  • 調試成本較高

建議落地順序

建立新 project / 新 workflow

不動舊版，先做 `whatsapp bot community redesign`，保留回滾空間。

先跑通最小聊天主線

Webhook → Normalize → Basic LLM Chain / AI Agent → Send WhatsApp Reply。

接入 Qdrant RAG

先只做查詢，再做寫入，避免一開始就把 memory policy 做複雜。

加入 Brave Search 和 image 支線

工具支線分開測，成功後再接回 Agent 或 LLM Chain。

補齊 profile、近期上下文和錯誤處理

最後再把個人語氣、群組規則、最近對話和 fallback 做漂亮。

我的推薦

先採用方案 B。它能保持現在 bot 的能力，又能把畫布變成清楚的社區節點設計。等 B 跑穩，再把部分工具升級成方案 C 的 Agent tools。
